# 背景
イントロンやその周辺のRNA二次構造がスプライシング効率に大きく影響する、というような研究が最近活発になっているらしい.

効率的なスプライシングが起きるようにどう DNA を設計すれば良いのか、という問題がある。

スプライシングについてだが、予測される二次構造の自由エネルギーが低いひどスプライシング効率が低下するというような話があるらしい。

だから、
 - イントロン配列周辺が構造を組まないようにするのは一つの重要な目的関数となる。

さらに、
 - イントロンの 5'/3' 末端のそれぞれ端から 3 塩基（intron）は特に塩基対を組んではならない、という目的関数もある

これはなぜかというと、
イントロンの 5' 末端の配列はある程度決まっていて（[CA]AG / GUAAGU のようになってる）、このうち イントロンの GU を含む数塩基は、スプライシングのための TNP が結合するために重要な領域となっていて、塩基対を組んでいて欲しくない。
また、イントロンの 3' 末端についても [CU]AG といった並びになることが多くて、それが 3' スプライス部位として厳密に認識される。だから、この部位が二次構造を強くつくっていると、因子が適切に認識できなくなる可能性があり、よくない。

# アプローチ、手法
そういうわけで、
 - イントロンは固定し、エクソン領域のコドンを同義置換で最適化する
 - 目的関数：
     - イントロンの5'/3'末端 3 塩基ずつの塩基対確率 $p(i, Stem)$ の合計をなるべく小さくする
     - イントロンの上流（5' 側）のエクソン領域 60 塩基 $x_{upstream}$ と intron 領域 $x_{intron}$ と下流（3'側）のエクソン領域 30 塩基 $x_{downstream}$ を結合した配列 $x = x_{upstream} + x_{intron} + x_{downstream}$ の partition function $Z$ を計算する。$- RT \log Z$ が小さいほど安定であるのだが、構造を組まない方が良いという話があるので、$-RT \log Z$ をなるべく大きくする


## 事前知識、定式化
配列は AUGC の 4 文字から成り立つ. 長さは L ということにしておく.
与えられるのは
 - 5utr
 - 3utr
 - main
    - exon: AUGC の大文字
    - intron: augc の小文字
